流動性結構

什麼是流動性？

在交易市場中，流動性 = 停損單的集中地 = 市場主力想要推動價格去的地方。

每一個停損單、每一個限價單都是「流動性」。當大量停損單堆積在某個價位附近，那裡就形成了「流動性池」（Liquidity Pool）。主力（做市商、機構）需要這些流動性來完成自己的大額訂單。

核心認知：「市場不是隨機移動的 — 價格會移動到流動性所在的地方。」價格像水一樣，會流向阻力最小、流動性最大的方向。

流動性池的位置

停損單（= 流動性）通常聚集在以下位置：

位置類型為什麼有流動性範例
前高/前低技術分析教科書教所有人把停損放在前高上方/前低下方BTC 前低 $94,200 → 大量空單停損在 $94,200 上方
等高/等低（Equal Highs/Lows）多次觸及同一價位 = 更多人在那裡設停損三次觸及 $100,000 不破 → 大量買入停損在 $100,050
整數關口心理價位，人類偏好在整數設停損和掛單$60,000、$65,000、$70,000
SuperTrend 線附近許多交易者以 ST 線作為停損參考位4H ST 在 $96,500 → 停損集中在 $96,200-$96,500
布林帶上下軌均值回歸交易者的停損區域BB 下軌 $62,000 → 停損在 $61,800
成交量缺口（FVG）快速移動造成的真空區 = 低阻力區域從 $95K 跳到 $97K 中間無成交量 = 公允價值缺口

如何在圖表上辨識流動性區域

視覺線索 1：多次觸及的水平線

  • 畫出被觸及 2 次以上但未突破的高/低點
  • 觸及次數越多 = 流動性越厚
  • 「第三次觸及通常會被突破」— 因為流動性太吸引人了

視覺線索 2：整數 + 密集成交

  • 看訂單簿深度（Order Book Depth）
  • 整數價位上方/下方通常有大量掛單
  • Heatmap（熱力圖）工具可直觀看到

視覺線索 3：Swing Point 密集區

  • 多個 Swing High 集中在相近價位
  • 代表多次嘗試突破 = 多次停損堆積
  • 這些區域是「磁鐵」— 價格終將被吸引過去

視覺線索 4：長期盤整區的邊界

  • 盤整時間越長 = 兩側堆積的流動性越多
  • 突破時流動性被觸發 = 加速移動
  • 盤整 > 7 天後的突破通常很劇烈

訂單簿深度（Order Book Depth）

訂單簿顯示所有未成交的限價單分布。它告訴你：

// 買方流動性（Bid Side）
$60,000: 150 BTC 掛單
$59,500: 80 BTC 掛單
$59,000: 320 BTC 掛單 ← 厚流動性牆

// 賣方流動性（Ask Side）
$61,000: 120 BTC 掛單
$61,500: 90 BTC 掛單
$62,000: 280 BTC 掛單 ← 厚流動性牆

// 解讀
- 大掛單可能是真實的支撐/壓力 → 也可能是「誘餌」(spoofing)
- 主力會在想買入前先掛大量賣單壓價，吃到低價後撤單
- 觀察掛單是否在價格靠近時消失 = spoofing 特徵
注意：訂單簿可以被操縱。大額掛單不一定代表真實意圖。主力經常使用「幽靈掛單」（Spoofing）— 在一側掛大量單來嚇退對手方，然後在另一側吃貨。真正的「流動性」是那些無法撤回的停損單。

為什麼價格會趨向流動性？

  1. 做市商的需求：做市商需要對手方才能成交大額訂單。100,000 BTC 的買入不可能一次在市場上完成，需要分批在有流動性的地方吃單。
  2. 停損觸發的連鎖反應：當價格觸及停損密集區，停損被觸發 = 市價單湧入 = 更多停損被觸發 = 瀑布效應。這為主力提供了充足的對手盤。
  3. 成交量需求：機構交易者需要足夠的成交量來建倉/出貨。流動性池 = 高成交量區 = 機構的「提款機」。
  4. 路徑最小阻力：價格傾向走向流動性較多（阻力較小）的方向。如果上方流動性 > 下方，價格更可能先往上走去「收割」上方的停損。

流動性核心法則

  • 流動性 = 停損單聚集地 = 主力的目標
  • 「市場移動到有流動性的地方」— 價格尋找停損
  • 前高前低、等高等低、整數關口 = 最明顯的流動性池
  • 觸及次數越多的水平線 = 流動性越厚 = 被突破的吸引力越大
  • 盤整越久 = 兩側堆積的流動性越多 = 突破越劇烈
  • 理解流動性位置 = 理解價格可能去哪裡

主力獵殺（Liquidity Hunt）

什麼是流動性獵殺？

流動性獵殺（Liquidity Hunt / Stop Hunt）是市場主力故意推動價格到停損密集區，觸發散戶的停損單，藉此獲得足夠的對手盤來完成自己的大額訂單，然後反向推動價格。

// 獵殺的基本模式
1. 識別：找到停損密集區（前高/前低/整數關口）
2. 推動：用相對少量的資金把價格推到停損區
3. 收割：停損被觸發 = 大量市價單湧入 = 主力的對手盤
4. 反轉：主力完成建倉/出貨後，價格迅速反向移動

// 散戶的視角
「明明走對了方向，就差一點被掃掉停損，然後就飛了」
→ 這就是典型的流動性獵殺

三種主要獵殺模式

掃流動性 Liquidity Sweep（流動性掃蕩）

定義：價格快速穿越前高/前低，觸發停損後立即反轉。通常在單根或極少根 K 線內完成。

特徵：

  • 突破前高/低後不持續，立即回撤
  • 留下長長的上/下影線（Wick / Shadow）
  • 收盤價回到突破前的區間內
  • 成交量在突破瞬間暴增然後驟降

範例：BTC 前低 $94,200，價格跌到 $93,900（穿越 $300），1 小時後回到 $95,500。$93,900-$94,200 這段就是「掃蕩」。

假突破 Fake Breakout（假突破）

定義：看似突破重要技術位但隨後失敗，價格反向移動。持續時間比 sweep 稍長（數小時到一天）。

特徵：

  • 突破時成交量不足（真突破應放量）
  • 突破後價格在突破位上方/下方猶豫不前
  • 後續 K 線實體逐漸縮小（動能衰竭）
  • 最終一根大陰/陽線吞沒突破 = 確認假突破

範例：BTC 盤整上軌 $68,000，突破到 $68,500 後橫盤 6 小時，量能萎縮，隨後暴跌到 $66,000。

SMC 獵殺 Smart Money Concept 完整獵殺

定義：完整的三步驟主力操作模式：掃流動性 → 位移（Displacement）→ 公允價值缺口（FVG）。

三步驟：

  1. 流動性抓取（Liquidity Grab）：價格穿越關鍵位觸發停損
  2. 位移（Displacement）：緊接著出現連續大實體 K 線反向移動（主力真正的方向）
  3. 公允價值缺口（Fair Value Gap / FVG）：位移過程中留下的價格真空區域 = 未來的回踔目標

範例：BTC 掃 $94,200 低點 → 出現連續 3 根大陽線從 $94,000 漲到 $97,000（位移）→ $95,000-$95,800 之間有 FVG → 後續回踔到 $95,400（FVG 中間）後繼續上漲。

如何辨識獵殺 vs 真突破

判斷要素是獵殺（假突破）是真突破
突破後動能突破後價格猶豫、停滯、回撤突破後價格加速遠離突破點
成交量突破瞬間有量但後續量能萎縮突破時放量且後續維持高量
K 線形態長影線、十字星、紡錘線大實體 K 線、無/短影線
收盤位置收盤回到突破前區間內收盤遠離突破點（> 0.5-1%）
時段低流動性時段（亞洲清晨、假日）高流動性時段（歐美交疊）
多時框配合僅 1H 突破、4H/日線未確認多時框同時突破
後續走勢形成反向大 K 線（吞沒）形成高低點不斷抬升/降低

用影線（Wick）辨識掃蕩

// 影線判讀規則

長下影線 + 小實體 = 下方流動性被掃蕩後買壓進場
長上影線 + 小實體 = 上方流動性被掃蕩後賣壓進場

// 有效掃蕩影線的標準
1. 影線長度 > 實體的 2 倍以上
2. 影線穿越了明確的流動性位（前高/前低/整數）
3. 收盤價回到流動性位的正確一側
4. 該 K 線成交量明顯高於前 5 根平均

// 操作含義
看到有效掃蕩影線 → 等下一根 K 線確認方向 → 順反轉方向入場
止損：影線最末端 + buffer

SMC 簡化模型：獵殺 → 位移 → FVG

Smart Money Concepts（聰明錢概念）的核心就是一個三步驟模型：

1
流動性抓取（Liquidity Grab）
價格快速穿越已知的流動性池（前高/前低）。特徵：長影線、快速回撤。
這一步的目的：觸發散戶停損 = 主力獲得對手盤
2
位移（Displacement）
掃蕩完成後，出現連續 2-3 根大實體 K 線向反方向移動。
這一步確認：主力的真正方向 = 位移的方向
3
公允價值缺口（Fair Value Gap / FVG）
位移過程中，第二根 K 線的高低點與第一、三根 K 線之間出現「缺口」（沒有重疊的價格區域）。
FVG 是未來的回踔目標：價格有 70%+ 的機率會回來填補這個缺口
實戰應用：在 FVG 中間位置掛限價單（Maker），止損在掃蕩影線的最末端。這是風報比極佳的進場策略：入場在 FVG（高機率觸及），止損在掃蕩極值（低機率再次觸及），目標是位移的延伸方向。

獵殺發生的時機

  • 亞洲時段尾聲 → 歐洲開盤：亞洲盤整的高/低點是獵殺目標
  • 重大數據公布前 15 分鐘：低流動性 + 恐慌 = 容易掃蕩
  • 週一開盤：掃蕩週五的高/低點
  • 假日薄盤：少量資金就能推動價格到流動性區
  • 長期盤整後：兩側流動性都很厚，先掃一側再往另一側走

散戶如何自保

  • 不要把停損放在「明顯」的位置（正好在前高/前低）
  • 停損 = 前高/前低 + 1-2 ATR buffer → 讓掃蕩夠不到你
  • 看到長影線掃蕩後的反轉 = 跟隨主力方向入場
  • 「被掃蕩」不代表判斷錯誤 — 只是停損設太緊
  • 學會在 FVG 回踔時入場，而不是在突破瞬間追單

低波動環境辨識

為什麼需要辨識低波動？

SuperTrend 等趨勢指標在低波動環境中會失靈。當市場進入極度壓縮期，趨勢策略會被反覆打臉，而網格策略則在這種環境中表現最佳。辨識低波動 = 知道何時該關掉趨勢系統、開啟網格系統。

低波動的五大辨識指標

1. ATR 壓縮

  • ATR(14) 低於過去 60 日平均的 50% 以下
  • 4H ATR 和日線 ATR 同時壓縮 = 雙重確認
  • ATR 壓縮持續 > 3 天 = 可以開始準備網格
  • 極端壓縮：ATR 創 30 日新低

2. 布林帶擠壓（BB Squeeze）

  • 上下軌距離收窄到歷史低位
  • BB 帶寬（BandWidth）< 過去 120 日最低值
  • K 線在 BB 中軌附近密集排列
  • 擠壓越極端 = 未來突破越猛烈

3. 成交量萎縮

  • 日成交量低於 20 日平均的 50%
  • 連續 3 天以上低量
  • BTC 24h 成交量創近期新低
  • 低量 = 無方向性 = 趨勢策略無效

4. 連續小實體 K 線

  • 連續 5 根以上 K 線實體 < ATR 的 30%
  • 紡錘線、十字星頻繁出現
  • 無明確的高低點推進
  • 視覺上 K 線像「螞蟻在爬」

5. 資金費率趨近零

  • 永續合約 Funding Rate 在 -0.005% ~ +0.005%
  • 代表多空力量完全均衡
  • 無方向偏差 = 無趨勢 = 適合網格
  • Funding 持續中性 > 3 天 = 強信號

低波動評分系統

指標條件分數
ATR(14) / ATR(60)< 50%+2
BB BandWidth120 日最低+2
24h 成交量 / 20MA< 50%+1
連續小實體 K 線> 5 根+1

判定標準：
  • 總分 >= 7：極度低波動 → 適合部署網格
  • 總分 5-6：中低波動 → 準備網格參數但暫不啟動
  • 總分 < 5：正常或高波動 → 使用趨勢策略（ST）

SuperTrend 為什麼在低波動中失靈？

// 正常環境
ATR(300) = $2,000 → ST 線離價格 $6,000（3 倍 ATR）
→ 微小波動觸發不了翻轉 → ST 信號可靠

// 低波動環境
ATR(300) 壓縮到 $800 → ST 線離價格僅 $2,400
→ 日常波動就可能觸及 ST 線 → 頻繁假翻轉

// 結果
1. ST 翻多 → 你做多 → 沒趨勢 → 價格回落到 ST 線 → 翻空
2. ST 翻空 → 你做空 → 沒趨勢 → 價格反彈到 ST 線 → 翻多
3. 反覆被來回打臉 → 連續虧損 → 心態崩潰

// 正確做法
辨識低波動 → 關閉 ST 交易系統 → 啟動網格策略

什麼造成低波動？

原因持續時間後續預期
重大事件前夕（CPI/FOMC/非農）1-3 天事件公布後劇烈波動 → 不適合網格
週末/假日2-3 天週一/假日後恢復正常波動
崩盤後盤整期1-4 週適合部署中期網格
Range-bound 市場數週至數月最佳網格環境（長期穩定）
暑假/年末淡季數週適合較寬參數的網格
監管不確定期不定可能突然劇烈波動，風控要嚴
關鍵判斷：低波動原因決定了是否適合開網格。「重大事件前」的低波動不適合開網格（因為事件會打破區間）。「崩盤後盤整」和「Range-bound 市場」才是理想環境。

低波動 → 高波動的轉折信號

當以下信號出現時，準備關閉網格、切回趨勢模式：

  • BB 帶寬開始擴張：上下軌開始分離 = 壓縮結束
  • 突然出現大實體 K 線：實體 > 前 10 根 K 線的平均 2 倍
  • 成交量暴增：24h 量 > 20MA 的 150%
  • Funding Rate 偏離零值：|FR| > 0.03% = 方向性出現
  • 價格突破 5 日 High/Low：脫離近期盤整區間

網格交易策略 SOP

什麼是網格交易？

網格交易（Grid Trading）是在一個價格區間內，等距設置一系列買入和賣出限價單。當價格在區間內波動時，自動低買高賣，每次填充（fill）都產生微利。

// 網格基本概念
區間上限 (Upper): $64,000
區間下限 (Lower): $60,000
格子數量 (Grids): 20 格
每格間距: ($64,000 - $60,000) / 20 = $200

// 運作邏輯
每個格子 = 一個掛單位
價格下跌到某格 → 觸發買入限價單
價格上漲到上一格 → 觸發賣出限價單
每次「買了又賣」= 賺一個格子的價差（$200）

// 網格的優勢
- 不需要預測方向（區間內雙向賺錢）
- Maker 單 = 零手續費或負手續費（掛單獎勵）
- 24/7 自動運作，不需盯盤
- 在低波動環境中持續產生現金流

何時部署網格 — 啟動條件

1
確認低波動環境
低波動評分 >= 7 分。ATR 壓縮 + BB 擠壓 + 成交量萎縮至少三項同時出現。
2
確認無即將到來的重大事件
查看經濟日曆：未來 48 小時內無 Tier 1 事件（CPI、FOMC、非農、央行利率決議）。
若有事件 → 不開網格，等事件結束後 24 小時再評估。
3
確認日線 ST 方向穩定
日線 ST 在同一方向持續 > 7 天（非剛翻轉）。
若日線 ST 剛翻轉 < 3 天 → 可能還在趨勢中，不適合網格。
4
確認價格在明確區間內
過去 5 天的價格都在一個 < 5% 的範圍內波動。
能明確畫出上軌和下軌。

網格參數設定 SOP

參數計算方式備註
區間上限近 5 日最高價 + 0.5 ATR(4H)留 buffer 避免太容易被突破
區間下限近 5 日最低價 - 0.5 ATR(4H)同上
格子間距0.3 ~ 0.5 倍 4H ATR間距太小 = 手續費吃掉利潤；太大 = 觸發次數少
格子數量（上限 - 下限）/ 間距通常 15-30 格為最佳
每格倉位總資金 / 格子數量確保所有買單都能被填充
總資金最多 20% 帳戶資金鐵律：永遠不超過 20%
// 計算範例

三種網格類型

中性網格（Neutral Grid）

適用：無方向偏好，純粹區間震盪

設定：當前價格在區間正中間。上下均勻分布格子。

特點：

  • 上半部 = 持有空單（每格賣出）
  • 下半部 = 持有多單（每格買入）
  • 不預判方向，雙向賺錢
  • 最大風險：價格離開區間

多頭偏向網格（Long-biased Grid）

適用：日線 ST 多，但短期盤整無方向

設定：只設買入格子（下方）。當價格回落時買入，上漲時分批賣出。

特點：

  • 只在下跌時建倉
  • 格子都設在當前價格下方
  • 如果價格一路上漲 = 沒有成交（錯過）
  • 如果價格下跌 = 分批接貨（低於市價）

空頭偏向網格（Short-biased Grid）

適用：日線 ST 空，但短期盤整反彈中

設定：只設賣出格子（上方）。當價格反彈時分批做空，下跌時分批平倉。

特點：

  • 只在上漲時建空倉
  • 格子都設在當前價格上方
  • 順日線大方向做空 = 安全邊際高
  • 如果趨勢反轉上漲 = 需要嚴格止損

Maker 費率優勢

網格策略的隱藏優勢：所有掛單都是限價單（Maker Order）。

交易所Maker 費率Taker 費率每格優勢
Binance（VIP 0）0.02%0.04%省 0.02% = 每 $10,000 省 $2
Binance（VIP 1+）0.00%0.035%零成本！每次 fill 都是純利潤
Bybit（VIP 1+）-0.005%（獎勵）0.025%掛單還倒賺手續費！
OKX0.02%0.05%省 0.03%
網格 + Maker = 印鈔機：當你的格子間距是 $320（≈ 0.5%），而 Maker 費率是 0%，那麼每次填充的 $320 就是純利潤。不需要像 Taker 一樣扣手續費。這就是網格在低波動中仍能獲利的核心原因 — 成本趨近於零。

網格的陷阱與限制

  • 單向行情 = 災難：如果價格直接穿越區間一路走掉，所有一側的單都被填充，另一側全空 = 持倉被套
  • 資金效率低：20% 帳戶資金分散在所有格子中，大部分時間只有少數格子在工作
  • 不適合高波動：ATR 突然擴大 = 區間被打破 = 網格失效
  • 心理陷阱：看到一側持倉越來越多（一邊倒）會焦慮，可能過早關閉
  • 事件風險：黑天鵝事件可能讓價格瞬間穿越整個網格 = 單向持倉爆滿

網格風險管理

每日審查清單

網格運行期間，每天必須檢查以下項目：

1
價格是否仍在區間內？
確認當前價格在網格上限和下限之間。若觸及邊界但未突破 = 暫時安全但需提高警覺。
2
日線 ST 方向是否改變？
日線 ST 翻轉 = 可能從盤整轉為趨勢 = 網格環境結束的信號。若翻轉方向 = 網格被套的方向 → 立即關閉。
3
ATR 是否開始擴張？
4H ATR 相比開網格時增加 > 50% = 波動率回升 = 區間可能被突破。準備關閉網格。
4
未來 24-48h 有無 Tier 1 事件？
CPI、FOMC、非農、央行利率決議。若有 → 事件前 4-12 小時關閉網格或暫停。
5
網格持倉是否過度偏向一側？
若所有買單都已成交但無賣單成交 = 持倉全在一側 = 接近被套邊緣。考慮主動減倉。
6
資金費率是否大幅偏離零？
|Funding Rate| > 0.05% = 市場開始有方向偏好 = 盤整可能結束。

殺手開關（Kill Switch）— 必須關閉網格的條件

立即關閉網格的觸發條件（任一即觸發）

  1. 日線 ST 翻轉：趨勢方向改變 = 盤整結束 → 立即平倉所有網格持倉
  2. 價格突破區間 > 1 ATR(4H)：不是只碰到邊界，而是遠離邊界超過一個 ATR = 確認離開區間
  3. Tier 1 事件即將在 24h 內發生：CPI/FOMC/非農 = 會造成劇烈波動打破區間
  4. 單方向連續行情 > 3%：連續數小時不回調的單向移動 = 趨勢啟動
  5. 成交量突然暴增 > 20MA 的 200%：巨量 = 大資金進場 = 方向性移動即將發生
  6. 連續 24h 無任何格子觸發：價格已離開活躍區域 = 網格形同虛設

網格被套：一邊倒持倉的處理

當價格移動到網格一側邊界，所有同方向的單都被填充：

情境 A：全部買單被填充

狀態：價格從上方跌到下方，所有買入格子都成交了。你持有大量多頭持倉，浮虧中。

決策流程：

  1. 價格仍在區間下限附近（未突破 > 1 ATR）→ 觀察，可能只是掃蕩下方流動性後反彈
  2. 價格突破下限 > 1 ATR 且日線 ST 翻空 → 認賠平倉，不要期待反彈
  3. 價格突破下限但日線 ST 仍多 → 平一半持倉，剩餘設止損在區間下限 - 2 ATR
情境 B：全部賣單被填充

狀態：價格從下方漲到上方，所有賣出格子都成交了。你持有大量空頭持倉，浮虧中。

決策流程：

  1. 價格仍在區間上限附近（未突破 > 1 ATR）→ 觀察，可能只是假突破
  2. 價格突破上限 > 1 ATR 且日線 ST 翻多 → 認賠平倉，趨勢已啟動
  3. 價格突破上限但日線 ST 仍空 → 平一半，剩餘設止損在區間上限 + 2 ATR

資金恢復策略

網格被套認賠後，如何恢復資金：

1
停止操作 24-48 小時
認賠後的情緒狀態不適合交易。給自己冷靜期。不要報復性加倉或立即開新網格。
2
重新評估環境
網格被突破 = 市場從盤整轉為趨勢。此時應切回 ST 趨勢策略，而非開新的網格。
3
降低下一筆的倉位
虧損後下一筆交易倉位減半。用小倉位慢慢恢復信心和資金。連續盈利 3 筆後再恢復正常倉位。
4
記錄並改進
為什麼被套？是忽略了殺手開關的信號？還是參數設太窄？記錄下來，下次避免。

網格資金管理鐵律

  • 最大 20% 帳戶資金：因為如果區間被打破，你可能虧損整個網格配置的 30-50%。20% * 50% = 10% 帳戶虧損 = 可承受
  • 永遠不用槓桿做網格：網格本身已經是全額佔用資金，加槓桿 = 爆倉風險
  • 單一資產最多一組網格：不要在同一個幣上開多組重疊網格
  • 所有網格加總不超過 40%：如果同時在 BTC 和 ETH 開網格 = 各 20% = 合計 40% 上限
  • 虧損觸及帳戶 5% 時強制關閉：不管其他條件如何，帳戶級別的止損是最後防線

網格風控總結

  • 網格是「有限環境」工具 — 只在確認的低波動區間使用
  • 每日審查是非談判性義務 — 跳過一天可能錯過殺手開關信號
  • 20% 資金上限是帳戶生存的保險 — 永遠不違反
  • 殺手開關 = 不猶豫的機械式行動 — 觸發就關，不期待
  • 被套認賠 = 正常成本 — 重要的是控制每次虧損 < 帳戶 5%
  • 網格只是工具箱中的一個工具 — 不是所有環境都適用
